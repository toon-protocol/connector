# Story 30.2: Implement Fire-and-Forget BLS Notification in PacketHandler

**Epic:** 30 - Per-Hop BLS Notification Pipeline
**Story Number:** 30.2
**Status:** Done

## Story Statement

**As a** connector operator,
**I want** intermediate connectors to fire a non-blocking notification to the local BLS when packets transit through,
**so that** every hop in the ILP path can observe, log, or trigger side-effects based on transiting packets — without affecting the critical forwarding path.

## Prerequisites

**Story Dependencies:** Story 30.1 (Add Per-Hop Notification Config and PaymentRequest Extension) — COMPLETED

**Epic Dependencies:**

- Epic 24 (Connector Library API — `LocalDeliveryClient`, `setLocalDeliveryHandler()`) — COMPLETED
- Epic 22 (Agent-Runtime Simplification — `SHA256(data)` fulfillment model) — COMPLETED

**Context:**
Story 30.1 added the `perHopNotification` config field to `LocalDeliveryConfig` and the `isTransit` flag to `PaymentRequest`. This story implements the actual fire-and-forget notification logic in `PacketHandler`, using the existing `LocalDeliveryClient.deliver()` for HTTP mode and `localDeliveryHandler` for in-process mode. The notification runs in parallel with BTP forwarding and cannot affect the critical path. Story 30.3 will add telemetry and integration tests.

## Acceptance Criteria

1. After routing table lookup resolves to a peer (not local), if `perHopNotification` is enabled and local delivery is configured:
   - HTTP mode: `this.localDeliveryClient.deliver(packet, sourcePeerId, { isTransit: true }).catch(...)` — no `await`
   - In-process mode: `this.localDeliveryHandler(request, sourcePeerId).catch(...)` — no `await`
2. The `PaymentRequest` sent includes `isTransit: true` for intermediate hops, `isTransit: false` (or omitted) for final-hop delivery
3. The BTP forwarding path (`forwardToNextHop`) is called unconditionally — the fire-and-forget notification runs in parallel
4. Failed notifications are logged at `debug` level (not `warn` or `error`) to avoid noisy logs
5. A catch handler prevents unhandled promise rejections (note: the epic spec uses `.catch(noop)` — this story intentionally upgrades to debug-level logging in the catch for better observability while remaining non-blocking)
6. Existing final-hop delivery behavior is completely unchanged
7. Unit tests verify: (a) notification fires when enabled, (b) notification does NOT fire when disabled, (c) forwarding succeeds even when notification throws, (d) `isTransit` flag is set correctly

### Implementation Pattern for Fire-and-Forget

In `handlePreparePacket()`, insert the following logic before the `forwardToNextHop()` call:

```typescript
// Fire-and-forget BLS notification for transit packets (per-hop notification)
const perHopEnabled = this.localDeliveryClient?.isPerHopNotificationEnabled() ?? false;
if (perHopEnabled) {
  if (this.localDeliveryHandler) {
    // In-process handler path (takes priority over HTTP)
    const transitRequest: LocalDeliveryRequest = {
      destination: packet.destination,
      amount: packet.amount.toString(),
      executionCondition: packet.executionCondition.toString('base64'),
      expiresAt: packet.expiresAt.toISOString(),
      data: packet.data.toString('base64'),
      sourcePeer: sourcePeerId,
      isTransit: true,
    };
    this.localDeliveryHandler(transitRequest, sourcePeerId).catch((err: unknown) => {
      this.logger.debug(
        {
          error: err instanceof Error ? err.message : String(err),
          destination: packet.destination,
        },
        'Per-hop notification failed (fire-and-forget, in-process)'
      );
    });
  } else if (this.isLocalDeliveryEnabled() && this.localDeliveryClient) {
    // HTTP client path
    this.localDeliveryClient
      .deliver(packet, sourcePeerId, { isTransit: true })
      .catch((err: unknown) => {
        this.logger.debug(
          {
            error: err instanceof Error ? err.message : String(err),
            destination: packet.destination,
          },
          'Per-hop notification failed (fire-and-forget, HTTP)'
        );
      });
  }
}
```

**Key design decisions:**

- Function handler takes priority over HTTP client (matches existing local delivery pattern at lines 960-1003)
- `.catch()` logs at `debug` level per AC 4 (not `warn` or `error`)
- The `forwardToNextHop()` call is placed unconditionally after this block (AC 3)
- `isTransit: true` is set in the `PaymentRequest`/`LocalDeliveryRequest` (AC 2)
