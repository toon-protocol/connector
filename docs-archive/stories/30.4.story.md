# Story 30.4: Remove XRP/Ripple and Aptos Settlement Support

**Epic:** 30 - Per-Hop BLS Notification Pipeline
**Story Number:** 30.4
**Status:** Done

## Story

**As a** connector maintainer,
**I want** XRP/Ripple and Aptos settlement code removed from the codebase,
**so that** the project focuses exclusively on EVM settlement, reducing complexity, test surface, and maintenance burden.

## Dev Notes

### Scope Overview

This is a large deletion story. ~45 files are deleted entirely, ~95+ files are modified across `packages/connector`, `packages/shared`, docs, and docker configs. The codebase shrinks significantly. The key principle: **delete aggressively, then fix compilation errors until `tsc --noEmit` passes.**

The scope spans:

- **packages/connector/src/** — settlement, core, config, http, wallet, security, observability, telemetry, explorer, cli, discovery, test-utils, test
- **packages/connector/explorer-ui/src/** — components, hooks, lib
- **packages/shared/src/** — shared telemetry and payment-channel types
- **packages/contracts-aptos/** — entire package deleted
- **docs/** and root-level files — architecture, README, docker-compose
