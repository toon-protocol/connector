# Story 30.1: Add Per-Hop Notification Config and PaymentRequest Extension

**Epic:** 30 - Per-Hop BLS Notification Pipeline
**Story Number:** 30.1
**Status:** Done

## Story Statement

**As a** connector operator,
**I want** a `perHopNotification` configuration field on `LocalDeliveryConfig` and an `isTransit` flag on `PaymentRequest`,
**so that** the connector can be configured to notify its local BLS at intermediate hops and the BLS can distinguish between transit notifications and final-hop delivery — without any behavioral changes yet.

## Prerequisites

**Story Dependencies:** None — this is the first story of Epic 30.

- Epic 24 (Connector Library API — `LocalDeliveryClient`, `setLocalDeliveryHandler()`) — COMPLETED
- Epic 22 (Agent-Runtime Simplification — `SHA256(data)` fulfillment model) — COMPLETED

**Context:**
This story adds types and configuration only. No behavioral changes are introduced. The `perHopNotification` field defaults to `false`, preserving full backward compatibility. Story 30.2 will implement the fire-and-forget notification logic using these types. Story 30.3 will add telemetry and integration tests.

### Environment Variable Loading Pattern

Environment variables for `LocalDeliveryConfig` are loaded in `ConnectorNode` constructor at `connector-node.ts` lines 189-201:

```typescript
const localDeliveryEnabled =
  resolvedConfig.localDelivery?.enabled || process.env.LOCAL_DELIVERY_ENABLED === 'true';
if (localDeliveryEnabled) {
  const localDeliveryConfig = {
    enabled: true,
    handlerUrl: resolvedConfig.localDelivery?.handlerUrl || process.env.LOCAL_DELIVERY_URL || '',
    timeout:
      resolvedConfig.localDelivery?.timeout ||
      parseInt(process.env.LOCAL_DELIVERY_TIMEOUT || '30000', 10),
    authToken: resolvedConfig.localDelivery?.authToken || process.env.LOCAL_DELIVERY_AUTH_TOKEN,
  };
  this._packetHandler.setLocalDelivery(localDeliveryConfig);
}
```

Add `perHopNotification` to this config object using nullish coalescing (`??`) instead of logical OR (`||`):

```typescript
perHopNotification:
  resolvedConfig.localDelivery?.perHopNotification ??
  (process.env.LOCAL_DELIVERY_PER_HOP_NOTIFICATION === 'true'),
```

**Important:** Use `??` (not `||`) here because `perHopNotification` is a boolean consumed inside the already-enabled block. With `||`, an explicit `false` in YAML config would fall through to the env var check, allowing the env var to override the explicit config value. The `??` operator correctly respects an explicit `false` and only falls through on `undefined`/`null`. (The existing `enabled` field uses `||`, but that's acceptable because `enabled` gates entry into the block — if it's `false`, the block is skipped entirely.)

The `perHopNotification` field must be passed through to `setLocalDelivery()` so that `PacketHandler` has access to it (needed in Story 30.2).

[Source: `packages/connector/src/core/connector-node.ts`, lines 189-201]
