## Dev Notes

### Why These Tests Are Being Deleted

**Fake acceptance tests:** The previous architectural review found that `production-acceptance.test.ts` only checks that classes can be imported. `performance-benchmark-acceptance.test.ts` only benchmarks OER serialization speed with synthetic data. `load-test-24h.test.ts` is `describe.skip` and uses `Math.random()` for synthetic failure injection. `multi-chain-settlement-acceptance.test.ts` uses `MockSettlementExecutor` with `Math.random() > failureRate`. None test real system behavior.

**Docker orchestration tests:** These test Docker Compose bring-up, container health checks, and port mapping. With embedded mode focus, connectors run in-process — Docker orchestration is infrastructure concern, not application behavior.

**Standalone-mode admin API tests:** `admin-channels-integration.test.ts` and `settlement-api-execution.test.ts` test HTTP admin API endpoints for channel management. In embedded mode, these APIs are not used — the direct `ConnectorNode` API replaces them.

**Redundant Base payment channel tests:** Three nearly-identical test files cover the same payment channel operations at slightly different transport layers. One parameterized test file is sufficient.

**Performance tests:** CPU profiling, memory profiling, latency/throughput benchmarks measure synthetic workloads (OER serialization in tight loops). They're useful as developer tools but shouldn't gate CI. If kept, they belong in a `scripts/benchmark/` directory, not the Jest test suite.

### wait-for.ts Utility Design

Replace the pervasive pattern:

```typescript
while (Date.now() - startTime < timeoutMs) {
  await new Promise((resolve) => setTimeout(resolve, 2000)); // FIXED 2s
}
```

With:

```typescript
// test/helpers/wait-for.ts
export async function waitFor(
  condition: () => Promise<boolean> | boolean,
  options?: { timeout?: number; interval?: number; backoff?: number }
): Promise<void> {
  const { timeout = 30000, interval = 100, backoff = 1.5 } = options ?? {};
  const start = Date.now();
  let delay = interval;
  while (Date.now() - start < timeout) {
    if (await condition()) return;
    await new Promise((resolve) => setTimeout(resolve, delay));
    delay = Math.min(delay * backoff, 5000);
  }
  throw new Error(`waitFor timed out after ${timeout}ms`);
}
```
