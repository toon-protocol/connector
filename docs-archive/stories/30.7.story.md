# Story 30.7: Embedded-Mode EVM Integration Test with Anvil

**Epic:** 30 - Per-Hop BLS Notification Pipeline
**Story Number:** 30.7
**Status:** Done

## Story

**As a** connector developer,
**I want** a comprehensive embedded-mode integration test that exercises the full EVM payment channel lifecycle against a local Anvil node,
**so that** I can verify that the connector's settlement, payment channels, and per-hop notification pipeline work end-to-end in embedded mode with real blockchain state.

## Acceptance Criteria

1. New integration test file: `test/integration/embedded-evm-settlement.test.ts`
2. Test uses embedded `ConnectorNode` instances (in-process, no Docker connectors) with `setPacketHandler()` for BLS mock (simpler `{ accept: true }` API; shares slot with `setLocalDeliveryHandler()`)
3. Test connects to local Anvil (started by `docker-compose-evm-test.yml` from Story 30.5) for real on-chain operations
4. Test exercises the full EVM payment channel lifecycle:
   - Open a payment channel between two embedded connectors (Token approval + channel open)
   - Fund the channel with AGENT tokens (via faucet or direct transfer)
   - Route ILP packets through the channel (Connector A → Connector B)
   - Verify off-chain claims are generated and exchanged via BTP
   - Close the channel and verify on-chain settlement
5. Per-hop notification pipeline verified: intermediate connector fires `isTransit: true` notification and `PER_HOP_NOTIFICATION` telemetry is emitted (proving Story 30.1-30.3 features work with real settlement)
6. All assertions use real blockchain state: `ethers.js` calls to verify on-chain balances, channel state, token transfers
7. Test is gated behind `EVM_INTEGRATION=true` environment variable (requires running Anvil)
8. Test completes in under 60 seconds (Anvil is fast — no block confirmations needed)
9. Test uses the `waitFor()` helper from Story 30.6 for health check polling (no fixed sleeps)
10. Test cleans up: channels closed, connectors stopped, BTP connections closed

### Embedded ConnectorNode Setup Pattern

```typescript
import { ConnectorNode } from '../../src/core/connector-node';
import type { ConnectorConfig } from '../../src/config/types';
import pino from 'pino';

const config: ConnectorConfig = {
  nodeId: 'connector-a',
  btpServerPort: 4101,
  deploymentMode: 'embedded',
  adminApi: { enabled: false },
  localDelivery: {
    enabled: true,
    perHopNotification: true,
  },
  settlementInfra: {
    enabled: true,
    rpcUrl: 'http://localhost:8545',
    registryAddress: '0xe7f1725E7734CE288F8367e1Bb143E90bb3F0512',
    tokenAddress: '0x5FbDB2315678afecb367f032d93F642f64180aa3',
    privateKey: '0x5de4111afa1a4b94908f83103eb1f1706367c2e68ca870fc3fb9a804cdab365a', // Anvil account 2
  },
  peers: [],
  routes: [],
  environment: 'development',
};

const logger = pino({ level: process.env.DEBUG ? 'debug' : 'silent', name: 'connector-a' });
const nodeA = new ConnectorNode(config, logger);
await nodeA.start();

// Register BLS handler via setPacketHandler (simpler than setLocalDeliveryHandler)
// PaymentHandler receives PaymentRequest with isTransit flag, returns { accept: true/false }
// Internally adapts to LocalDeliveryHandler (fulfillment computed automatically)
nodeA.setPacketHandler(async (request) => {
  // Mock BLS that accepts payments; request.isTransit is true for transit notifications
  return { accept: true };
});

// Register peer (all fields required: id, url, authToken)
await nodeA.registerPeer({
  id: 'connector-b',
  url: 'ws://localhost:4102',
  authToken: 'test-secret-ab',
  routes: [{ prefix: 'g.connectorB' }],
});

// Send a packet
const result = await nodeA.sendPacket({
  destination: 'g.connectorB.receiver',
  amount: 1000n,
  executionCondition,
  expiresAt,
  data,
});
```

### Payment Channel Lifecycle (EVM)

The `PaymentChannelSDK` at `src/settlement/payment-channel-sdk.ts` provides:

1. **Open Channel**: `openChannel(participant2, tokenAddress, settlementTimeout, initialDeposit)` → `{ channelId, txHash }`
2. **Deposit**: `deposit(channelId, tokenAddress, amount)` — handles ERC20 approval internally
3. **Sign Balance Proof**: `signBalanceProof(channelId, nonce, transferredAmount, lockedAmount?, locksRoot?)` → EIP-712 signature string
4. **Close Channel**: `closeChannel(channelId, tokenAddress, balanceProof, signature)` — initiates challenge period
5. **Cooperative Settle**: `cooperativeSettle(channelId, tokenAddress, myProof, mySig, theirProof, theirSig)` — bypasses challenge
6. **Query State**: `getChannelState(channelId, tokenAddress)` → `ChannelState` (status: opened, closed, settled)

Key types from `@crosstown/shared`:

- `BalanceProof`: `{ channelId, nonce, transferredAmount, lockedAmount, locksRoot }`
- `ChannelState`: `{ channelId, participants, myDeposit, theirDeposit, status, ... }`

### Refactoring Performed

Three bugs found and fixed during live testing:

- **File**: `test/integration/embedded-evm-settlement.test.ts`
  - **Change**: Added unique `healthCheckPort` per connector (HEALTH_PORT_A/B/C)
  - **Why**: All 3 embedded connectors competed for default port 8080, causing EADDRINUSE
  - **How**: Added port constants offset +10/+11/+12 from BASE_PORT

- **File**: `test/integration/embedded-evm-settlement.test.ts`
  - **Change**: Added `sdkB.getChannelState(channelId, TOKEN_ADDRESS)` cache warmup before balance proof test
  - **Why**: `sdkB.verifyBalanceProof()` and `sdkB.signBalanceProof()` failed because `tokenNetworkCache` was empty (only sdkA opened the channel)
  - **How**: Calling `getChannelState` populates the SDK's internal tokenNetworkCache via `getTokenNetworkContract()`

- **File**: `test/integration/embedded-evm-settlement.test.ts`
  - **Change**: Added SDK listener cleanup and async drain in afterAll
  - **Why**: Jest warning about open handles from WebSocket connections
  - **How**: Call `sdkA.removeAllListeners()` and `sdkB.removeAllListeners()` + 100ms drain

- **File**: `packages/faucet/Dockerfile`
  - **Change**: Changed `npm ci` to `npm install`
  - **Why**: Faucet package has no `package-lock.json`, causing Docker build failure
  - **How**: `npm install` works without lock file

- **File**: `packages/faucet/src/index.js`
  - **Change**: Wrapped wallets in `ethers.NonceManager` and added request serialization lock
  - **Why**: Concurrent faucet requests caused nonce race conditions on Anvil (NONCE_EXPIRED errors)
  - **How**: `NonceManager` tracks pending nonces; promise-based lock serializes request processing
